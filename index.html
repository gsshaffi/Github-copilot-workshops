Welcome — GitHub Copilot for Tech Teams

An interactive, hands-on workshop for Connells Group engineering teams • 60 minutes

What is this session? A practical, hands-on workshop where you will use GitHub Copilot to write real code, generate infrastructure, debug issues, and automate routine tasks — all within VS Code. The goal is for you to leave with Copilot installed and an understanding of how it saves time in your daily work.

Two Sessions, One Goal

This guide covers both sessions. Look for the track labels to find content tailored to your team:

Infra Track

Audience: Infrastructure, Platform, DevOps, SRE teams

Focus areas:

  • Terraform & Bicep generation
  • PowerShell scripting & automation
  • CI/CD pipeline YAML
  • Troubleshooting & incident response

Dev Track

Audience: Software developers, product engineering, QA

Focus areas:

  • Writing features & refactoring code
  • Unit tests & debugging
  • Documentation & code review
  • Working with unfamiliar codebases

Code Completions (Ghost Text)

Both Tracks • The core feature — AI-powered autocomplete on steroids

How it works: As you type, Copilot analyses your current file, open files, and comments, then suggests entire lines or blocks of code as grey “ghost text.” You press Tab to accept, Esc to dismiss, or Alt+] to cycle through alternatives.

Key Shortcuts

ActionShortcut
Accept suggestionTab
Dismiss suggestionEsc
Next suggestionAlt+]
Previous suggestionAlt+[
Accept word-by-wordCtrl+Right
Trigger suggestions manuallyAlt+\

The Secret: Comments Drive Completions

Copilot reads your comments as intent. Writing a clear comment before a function is the single best way to get accurate suggestions:

example.py # Calculate the monthly mortgage repayment amount # given the principal, annual interest rate, and term in years # using the standard amortisation formula def calculate_monthly_repayment(principal, annual_rate, term_years): monthly_rate = annual_rate / 12 / 100 num_payments = term_years * 12 if monthly_rate == 0: return principal / num_payments payment = principal * (monthly_rate * (1 + monthly_rate)**num_payments) / ((1 + monthly_rate)**num_payments - 1) return round(payment, 2)

The grey italic text is what Copilot suggests automatically after you write the comment and function signature.

Next Edit Suggestions (NES)

A newer feature — when you edit one part of a file, Copilot proactively suggests the next logical edit you need to make. You’ll see a lightbulb or ghost text at a different location in the file. Press Tab to accept and jump there.

✍️ Try It: Comment-Driven Completions

  1. Create a new file: mortgage.py (or mortgage.cs / mortgage.ts — any language you like)
  2. Type a comment: # Validate a UK postcode format and return true if valid
  3. Press Enter and start typing def validate_postcode(
  4. Watch the ghost text appear — press Tab to accept
  5. Now add another comment: # Calculate stamp duty for a property purchase in England
  6. Let Copilot generate the full function

Discussion: How close was the output? How much editing did you need? That editing part is important — Copilot drafts, you refine.

✍️ Try It: Repetitive Patterns

Copilot excels at continuing patterns. Try this:

  1. Create a new file and type a few lines of a mapping or configuration
  2. For example, start defining a dictionary of UK regions to office codes
  3. After 2–3 entries, Copilot will predict the pattern and fill the rest

Copilot Chat

Both Tracks • Your AI pair programmer in the sidebar

Open Copilot Chat: Ctrl+Alt+I — or click the Copilot icon in the sidebar. You can ask questions, reference files, and get code suggestions in a conversational format.

What Can You Ask?

Explain & Understand

  • “Explain this function”
  • “What does this regex do?”
  • “What are the security implications of this code?”
  • “Summarise the changes in this file”

Generate & Create

  • “Write a function that validates an email address”
  • “Create a SQL query to find duplicate records”
  • “Generate a Terraform module for an Azure App Service”
  • “Create a PowerShell script to check disk space”

Context Is King: Using @ References

The more context you give Copilot, the better the answer. Use @ to reference specific context:

ReferenceWhat it doesExample
@workspaceSearches your entire workspace for relevant code“@workspace where is the database connection configured?”
#fileReferences a specific file“Explain #file:appsettings.json”
#selectionReferences your currently selected code“Refactor #selection to use async/await”
#terminalLastCommandReferences the last terminal command output“Explain this error #terminalLastCommand”

Slash Commands

Quick shortcuts for common tasks:

CommandWhat it does
/explainExplain the selected code or file
/fixPropose a fix for problems in the selected code
/testsGenerate unit tests for the selected code
/docGenerate documentation for the selected code
/clearStart a fresh chat conversation

✍️ Try It: Chat with Your Code

  1. Open the file you created in the last exercise (or any code file from your projects)
  2. Open Copilot Chat (Ctrl+Alt+I)
  3. Type: “Explain what this file does and suggest improvements”
  4. Now select a function and type: “/tests generate unit tests for this function”
  5. Try: “Rewrite this to handle edge cases like empty input and negative numbers”

✍️ Try It: The “Unfamiliar Codebase” Scenario

One of the biggest time-savers — using Copilot to understand code you did not write:

  1. Open any file from a project you are less familiar with
  2. Ask: “@workspace give me an overview of this project’s architecture”
  3. Ask: “@workspace where is the authentication logic?”
  4. Ask: “@workspace how does the data flow from the API to the database?”

Discussion: How long would it have taken to answer those questions by reading the code manually?

Inline Chat & Quick Fixes

Both Tracks • Edit code without leaving the editor

Open Inline Chat: Select code (or place your cursor) and press Ctrl+I. A small prompt box appears directly in the editor — type your instruction and Copilot edits the code in place, showing a diff you can accept or reject.

When to Use Inline Chat vs Sidebar Chat

Use Inline Chat When...Use Sidebar Chat When...
You want to edit a specific block of codeYou want to generate something from scratch
You want to see the diff immediatelyYou want to have a longer conversation
Quick refactor, rename, fixArchitecture questions, explanations
Adding error handling to existing codeComparing approaches, multi-file tasks

Quick Fix Integration

When VS Code shows a squiggly underline (error or warning), hover over it and look for the lightbulb (Ctrl+.). Copilot adds AI-powered fix suggestions alongside the standard VS Code quick fixes:

  • “Fix using Copilot” — Copilot analyses the error and proposes a fix
  • “Explain using Copilot” — explains what the error means and why it occurred

✍️ Try It: Inline Refactoring

  1. Open your mortgage.py file (or create a function with some messy code)
  2. Select a function, press Ctrl+I
  3. Type: “Add input validation and meaningful error messages”
  4. Review the diff — press Enter to accept or Esc to reject
  5. Now select the same code and press Ctrl+I again
  6. Type: “Add type hints and a docstring”

✍️ Try It: Fix an Error

  1. Deliberately introduce a bug (wrong variable name, missing import, off-by-one error)
  2. Wait for the red squiggly underline to appear
  3. Hover and click the lightbulb → choose “Fix using Copilot”
  4. See how Copilot diagnoses and fixes the issue

Agent Mode

Both Tracks • Multi-step, multi-file tasks — Copilot plans and executes

What makes Agent Mode different? In normal chat, Copilot answers questions. In Agent Mode, Copilot takes action — it can create files, edit multiple files, run terminal commands, and iterate on errors. It plans a sequence of steps and executes them, asking for your approval at each stage.

How to Activate Agent Mode

  • 1
    Open Copilot Chat (Ctrl+Alt+I)
  • 2
    At the top of the chat panel, switch the mode dropdown from “Ask” to “Agent”
  • 3
    Type your request — describe the outcome, not individual steps
  • 4
    Copilot will propose a plan. Review each step and approve edits as they come

What Agent Mode Can Do

Create & Scaffold

  • Create a whole project structure with multiple files
  • Generate a REST API with models, routes, and tests
  • Scaffold IaC for an environment with networking, compute, and storage

Edit & Fix

  • Refactor across multiple files simultaneously
  • Fix a failing build by reading the error and editing the source
  • Run tests, see failures, and iterate until they pass
Free Tier Note: Agent mode uses your 50 monthly chat messages. Each “turn” in the conversation counts as one message. A multi-step task might use 3–5 messages. Use them wisely during the workshop — aim for 2–3 agent exercises.

✍️ Try It: Build Something with Agent Mode

Infra teams:

AGENT PROMPT Create a Terraform module that deploys an Azure resource group, a virtual network with two subnets (web and data), and an NSG that allows HTTPS inbound on the web subnet only. Include variables for location, environment name, and address space. Output the resource group name and subnet IDs.

Dev teams:

AGENT PROMPT Create a simple REST API in C# (.NET 8 minimal API) with: - A GET /properties endpoint that returns a list of properties - A GET /properties/{id} endpoint that returns a single property - A Property model with fields: Id, Address, Price, Bedrooms, Status - In-memory data store with 5 sample properties - Basic input validation and error handling Include a .http file to test the endpoints.
  1. Switch to Agent mode in the chat dropdown
  2. Paste or type the prompt above (pick your track)

  4. Review each proposed change before accepting
  5. Discussion: How long would this have taken from scratch?

Infrastructure as Code: Terraform & Bicep

Infra Track • Generate, explain, and refactor IaC

Why Copilot is a game-changer for IaC: Terraform and Bicep involve a huge amount of boilerplate, provider-specific syntax, and best-practice configuration that is hard to memorise. Copilot knows the patterns and generates compliant, parameterised code from natural language descriptions.

Terraform Exercises

✍️ Exercise 1: Generate a Module from a Comment

  1. Create a file: main.tf
  2. Type this comment at the top:
main.tf # Azure Storage Account with: # - Standard LRS replication # - Blob container called "backups" with private access # - Lifecycle rule to move blobs to cool tier after 30 days # - Tags: environment and managed_by
  1. Press Enter and watch Copilot generate the Terraform resources
  2. Accept with Tab, then keep pressing Enter — Copilot will continue generating related blocks

✍️ Exercise 2: Explain Existing IaC

Paste or open a complex Terraform file from your projects, then in Chat:

CHAT PROMPT Explain this Terraform configuration. What resources does it create, what are the dependencies between them, and are there any security concerns or best-practice violations?

Bicep Exercises

✍️ Exercise 3: Convert ARM to Bicep

If you have an ARM template, paste it into chat and ask:

CHAT PROMPT Convert this ARM template to Bicep. Use parameter decorators, meaningful resource names, and add descriptions to all parameters.

✍️ Exercise 4: Add to Existing IaC

Open your generated Terraform/Bicep and use inline chat (Ctrl+I):

  • “Add a diagnostic setting that sends logs to a Log Analytics workspace”
  • “Add a private endpoint for this storage account”
  • “Add outputs for the connection string and primary key”

Pipelines & YAML

Infra Track • CI/CD configuration at speed

YAML is Copilot’s sweet spot. CI/CD files are highly structured and repetitive. Copilot has seen millions of pipeline definitions and generates accurate, well-structured YAML with the right indentation (which matters a lot in YAML!).

✍️ Exercise 1: Generate a Pipeline

  1. Create a file: azure-pipelines.yml (or .github/workflows/build.yml)
  2. Type a comment at the top:
azure-pipelines.yml # Azure DevOps pipeline that: # - Triggers on pushes to main and develop # - Uses a .NET 8 build agent # - Restores NuGet packages # - Builds the solution # - Runs unit tests with code coverage # - Publishes the artifact for deployment
  1. Let Copilot generate the pipeline
  2. Use inline chat to: “Add a deployment stage to an Azure App Service with approval gate”

✍️ Exercise 2: Docker + Compose

CHAT PROMPT Create a Dockerfile for a .NET 8 web API that uses multi-stage build for a smaller image. Then create a docker-compose.yml that includes the API, a SQL Server container, and a Redis cache container with appropriate networking and environment variables.

✍️ Exercise 3: Explain and Fix a Pipeline

Open an existing pipeline YAML file and ask:

  • “/explain what does this pipeline do step by step?”
  • “Are there any security issues with this pipeline? (e.g. hardcoded secrets, excessive permissions)”
  • “Optimise this pipeline for faster build times”

Tests & Debugging

Dev Track • The tasks nobody wants to do manually

Writing tests is one of Copilot’s highest-value use cases. Developers consistently report that test generation is where they save the most time. Copilot understands your code structure and generates meaningful test cases including edge cases you might miss.

Generating Tests

✍️ Exercise 1: Unit Tests from Code

  1. Open the code you generated in the previous exercises
  2. Select a function or class
  3. Right-click → “Copilot” → “Generate Tests” (or type /tests in chat)
  4. Review the generated tests — note how Copilot includes edge cases

Alternatively, in Chat:

CHAT PROMPT Generate xUnit tests for #file:ValuationCalculator.cs Include: - Happy path tests for each method - Edge cases: zero values, negative numbers, null inputs - Boundary tests for postcode validation - Use descriptive test names following the Given_When_Then pattern

✍️ Exercise 2: Debug a Failing Test

  1. Run your tests (some may fail if the generated code needs tweaking)
  2. Copy the test failure output
  3. In Chat, type: “This test is failing with this error: [paste error]. Explain why and fix it.”
  4. Or use #terminalLastCommand: “Explain why this test failed #terminalLastCommand and fix the code”

✍️ Exercise 3: Explain a Stack Trace

Got a production error? Paste the stack trace into Chat:

CHAT PROMPT Explain this stack trace in plain English. What went wrong, what is the root cause, and what are the likely fixes? [paste your stack trace here]

Tips & Best Practices

Get the most out of GitHub Copilot

The “Copilot Mindset”

Think of Copilot as a very fast junior developer. It is incredibly productive at drafting code, but you are the senior engineer who reviews, refines, and takes responsibility for the output. The best results come from clear instructions and active review—not blind acceptance.

Top Tips for Better Results

1. Write Good Comments First

Comments are prompts. A clear, specific comment produces dramatically better completions than starting to type code directly.

2. Open Relevant Files

Copilot uses your open tabs as context. If you are building a new endpoint, have the model, existing endpoints, and tests open in tabs.

3. Be Specific in Prompts

Instead of “create a function,” say “create a C# method that takes a postcode string, validates it against the UK format, and returns a bool.”

4. Iterate, Do not Start Over

If the first output is 80% right, refine with follow-up prompts rather than starting a new conversation. Context builds up.

5. Use @ References Aggressively

@workspace, #file, #selection — the more context you provide, the more accurate the output.

6. Review Before Accepting

Always read the suggestion. Watch for hallucinated APIs, incorrect business logic, or security issues. You own the code.

What Copilot Is Not Good At

  • It does not know your business rules — it can guess, but you must verify domain logic
  • It can hallucinate APIs — especially for newer or less common libraries
  • It does not replace understanding — use it to accelerate work you understand, not to write code you cannot read
  • Security-critical code — always have a human review security-sensitive paths
  • It has a knowledge cutoff — very new frameworks or breaking changes may not be reflected

Privacy & Security

On the Free tier:
  • Code snippets are sent to GitHub’s servers for processing — they are not used to train models
  • Prompts and suggestions are not retained after the response is returned (on Business/Enterprise plans)
  • Do not paste secrets, credentials, or sensitive personal data into prompts
  • For the organisation: GitHub Copilot Business/Enterprise gives you data exclusion policies, audit logs, and IP indemnity
