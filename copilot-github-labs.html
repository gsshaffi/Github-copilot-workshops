GitHub Copilot — Hands-On Lab Guide

Practical exercises to accompany the GitHub Copilot Workshop • Connells Group

How to use this guide. This lab guide contains step-by-step exercises. Each exercise tells you exactly which file to open, what to do with Copilot, and what the expected outcome looks like. Work through your track’s exercises in order, or pick the ones that interest you most.

Lab Repos

You’ll need two repos. Your facilitator will share the GitHub URL — clone them before starting.

Infra Lab

Terraform, PowerShell scripts, CI/CD pipelines, Dockerfiles, and runbook templates.

Folder: infra-lab/

Dev Lab

C# / .NET 8 property management API. Models, services, controllers, SQL, legacy code, and tests.

Folder: dev-lab/

Difficulty Levels

  • Starter — Follow the steps, let Copilot do most of the work. Great for your first exercises.
  • Intermediate — Requires some prompt crafting. You may need to guide Copilot with context.
  • Advanced — Open-ended challenges. Multiple approaches, multiple files. Real-world complexity.

Copilot Quick Reference

Feature How to Trigger Best For
Completions Just start typing or write a comment Filling in code, completing patterns
Chat Panel Ctrl+Shift+I Questions, explanations, generating blocks of code
Inline Chat Ctrl+I Quick edits in-place — refactor, fix, explain
Agent Mode Chat panel → switch to Agent Multi-file tasks, scaffolding, complex changes

Clone & Open the Repos

Get the lab repos into VS Code so you’re ready for the exercises

Step-by-Step

  1. 1
    Open a terminal (Ctrl+`) in VS Code.
  2. 2
    Clone the lab repo (your facilitator will provide the URL):
    Terminalgit clone https://github.com/YOUR-ORG/github-copilot-labs.git cd github-copilot-labs
  3. 3
    Open the folder in VS Code: File → Open Folder and select github-copilot-labs.
  4. 4
    Check Copilot is active — look for the Copilot icon in the bottom-right status bar. It should show a tick mark, not a warning.
  5. 5
    Navigate to your track’s folder:
    • Infra Track → open infra-lab/
    • Dev Track → open dev-lab/
No GitHub account yet? Sign up at github.com and activate Copilot Free from your profile settings → Copilot. It takes less than 2 minutes and gives you 2,000 completions + 50 chat messages per month.

Infra Track Terraform Completion

Use Copilot completions and chat to finish a Terraform configuration

Exercise 3.1 Starter

Complete the Variables File

  1. Open infra-lab/terraform/variables.tf
  2. Place your cursor after the last variable block
  3. Type variable "data_subnet_prefix" and press Enter
  4. Copilot should suggest the full variable block — press Tab to accept
  5. Continue for storage_account_tier, storage_replication_type, and common_tags

Expected outcome: 4 new variable blocks with sensible defaults and descriptions, matching the TODO comments.

Exercise 3.2 Intermediate

Complete main.tf with Missing Resources

  1. Open infra-lab/terraform/main.tf
  2. Read through the existing resources (resource group, VNet, web subnet, web NSG)
  3. Place your cursor at the # TODO: Add data subnet comment
  4. Use Copilot completions or inline chat (Ctrl+I) to generate each missing resource
  5. Complete all the TODOs: data subnet, data NSG, storage account, and key vault
Inline Chat PromptAdd an azurerm_subnet for the data tier using the data_subnet_prefix variable, associated with the existing VNet

Expected outcome: 4 additional resources that reference existing variables and follow the same naming conventions.

Exercise 3.3 Intermediate

Build the Networking Module

  1. Open infra-lab/terraform/modules/networking/main.tf
  2. Read the comments describing what the module should contain
  3. Open Copilot Chat (Ctrl+Shift+I) and try:
Chat PromptBased on the comments in this file, generate the full Terraform networking module with variables, the VNet resource, two subnets, two NSGs, and outputs

Expected outcome: A complete module with variables.tf, main.tf, and outputs.tf content.

Exercise 3.4 Starter

Generate Outputs

  1. Open infra-lab/terraform/outputs.tf
  2. Type output "resource_group_name" and let Copilot complete it
  3. Continue accepting suggestions — Copilot should generate outputs for all resources in main.tf

Expected outcome: Output blocks for resource group name, VNet ID, subnet IDs, storage account name, and Key Vault URI.

  • Completed variables.tf
  • Completed main.tf with all resources
  • Built the networking module
  • Generated outputs.tf

Both Tracks Agent Mode Challenge

A capstone exercise using Copilot Agent Mode for a multi-file task

What is Agent Mode? Agent Mode can plan and execute multi-step tasks across multiple files. It reads your codebase, proposes a plan, creates/edits files, and runs terminal commands — all from a single prompt. It’s like pair-programming with an AI colleague who can actually type.
Challenge A Infra Track Advanced

Build a Monitoring Dashboard Config

Open Copilot Chat, switch to Agent mode, and give this prompt:

Agent Mode PromptUsing the infra-lab repo as context, create: 1. A Terraform module for Azure Monitor alerts (CPU > 80%, memory > 90%, disk > 85%) 2. An Azure Dashboard definition (JSON) showing the health check metrics 3. An alert action group that sends email to the ops team 4. Update the main.tf to reference the new monitoring module Put the module in infra-lab/terraform/modules/monitoring/

Watch Agent Mode plan, create the module directory, write the Terraform files, and update existing config.

Challenge B Dev Track Advanced

Implement Search & Statistics

Open Copilot Chat, switch to Agent mode, and give this prompt:

Agent Mode PromptUsing the dev-lab repo as context: 1. Add Search and GetStatistics methods to IPropertyService 2. Implement them in PropertyService with LINQ queries 3. Add the search and stats endpoints to PropertyController 4. Wire up the endpoints in Program.cs 5. Generate xUnit tests for both new features Keep the existing code style and patterns.

Watch Agent Mode trace through the interface, service, controller, and tests to implement the full feature.

What to observe: Notice how Agent Mode reads existing files to match patterns, proposes a plan before acting, edits multiple files in sequence, and occasionally runs commands. This is the workflow that saves the most time on real-world feature work.
